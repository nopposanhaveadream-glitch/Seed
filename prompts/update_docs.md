# ドキュメント更新依頼テンプレート

設計議論の結果を既存ドキュメントに反映する際の依頼テンプレート。

## このテンプレートの使い方

以下の形式で具体的な内容を埋めて、Claude Codeに渡す:

```
## 背景
[何の議論を受けた更新か]

## 変更対象ファイル
- [ファイルパス]

## 変更内容
[具体的な変更箇所と追記内容。可能な限り現在の内容と追記後の内容を明示]

## してはいけないこと
- コードファイル（core/, phase0/, simulation/, tests/）は一切変更しない
- 確定パラメータ（v5最終版）は一切変更しない
- 他のドキュメントは変更しない（指定したファイルのみ）

## 完了報告
1. 変更したファイルのリスト
2. 差分サマリー
3. 第一原則への影響チェック（ドキュメントのみの変更であることを明示）
```

## よくある更新パターン

### 1. 設計原則の追記
対象: `docs/PRINCIPLES.md`
- 既存の原則番号を変えずに新規原則を追加
- または既存原則の文言を修正（この場合は必ず「なぜ変更するか」を明記）

### 2. ROADMAP.mdへの追記
対象: `docs/ROADMAP.md`
- 新しいフェーズの追加
- 5月上旬アジェンダへの追加
- 変更履歴セクションへの記録

### 3. 長期論点の追加
対象: `docs/future_agenda.md`
- 番号を振って末尾に追加
- 「論点」「背景」「設計上の問い」「タイミング」を含める

### 4. シミュレーション履歴の追加
対象: `docs/simulation_results.md`
- 履歴セクションに新項目を番号付きで追加
- 最終更新日を更新

### 5. README.mdのステータス更新
対象: `README.md`
- Phase 1のチェックリスト更新
- フェーズC中間観測セクションの更新

## 原則

- 既存の記述を矛盾なく保つ
- 変更は必要最小限に
- コード変更と混同しない
- フェーズC中のパラメータ凍結を維持
