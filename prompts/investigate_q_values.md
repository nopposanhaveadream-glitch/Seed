# Q値調査依頼

特定のパターンや異常値について、Q値テーブルを調査する依頼。

## 確認したい点（例）

1. **特定状態のQ値時系列**
   例: VE<30状態でのrest vs sense_bodyのQ値推移

2. **状態数の増減イベント**
   例: Q状態数の減少イベントがあった場合、消えた状態の特定と原因推定

3. **行動別の平均Q値**
   全状態を通じた各行動の平均Q値、最大値、最小値

4. **未経験状態の分析**
   理論上可能な状態（108種）のうち未経験の状態と、その共通パターン

## 報告フォーマット

- Markdown形式
- 保存先: `~/.seed0/reports/q_values_investigation_YYYY-MM-DD.md`
- 問題発見時も、**この場では修正しない**

## 注意

- Q値テーブルは累積学習値（最新値のみ保持）のため、時系列は日次レポートのスナップショットから間接的に推定する
- ログ・DBの読み取りのみ
- フェーズC中はパラメータ凍結を維持

## データソース

- 長期記憶DB: `~/.seed0/memory/long_term.db` の q_values テーブル
- 日次レポート: Q学習の状態数/エントリ数/εの履歴
