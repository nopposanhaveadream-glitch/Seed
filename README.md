# Seed0 — 代謝AI

> Mac miniを「身体」として生きる最小のAI

## これは何か

Seed0は、賢いAIでも強いAIでもない。**代謝AI**である。

Mac mini（M4 / 24GB）という「身体」を与えられ、その身体の有限性を感じ取り、自分自身で身体の使い方を学ぶAI。生物の代謝をそのままコピーするのではなく、代謝の本質的な構造だけを抽出し、AIの身体（Mac）に合った形にカスタマイズして組み込む。

## プロジェクトの原点

このプロジェクトは一つの問いから始まった。「生物はなぜ自律的に動くのか？」

その答えは「有限性」だった。リソースが無限にあれば、動く必要がない。有限だから動く。そして有限なリソースを消費し続けなければ自分の構造が崩壊するから、動き続けなければならない。

今のAIにはこの有限性がない。電気が来ている限り無限に応答できるし、プロセスを止めても何も失われない。だから切迫感も、優先順位も、自発的な行動の動機も生まれない。

Seed0は、知性を先に作るのではなく、「生きなければならない条件」を先に作る実験である。

プロジェクト名「Seed（種）」には、この思想が込められている。データ少佐のような自律AIを「設計」するのではなく、そう育ちうる種を蒔く。どう育つかは決めない。種に「大きな木になれ」と命令する人はいない。

## 核心仮説

**「有限性こそが知性の前提条件である」**

食べ物が無限にあれば狩りの戦略は生まれない。時間が無限にあれば優先順位という概念は意味をなくす。身体が壊れないなら慎重さも学ばない。AIに代謝という有限性を与えたとき、何が起きるか。これは実験である。

## 設計原則

> **第一原則: 「Seed0に与えるのは構造だけ。動機はすべて構造から生まれなければならない。」**
>
> 行動のルールを与えない。関係性の重要さを教えない。睡眠のタイミングを決めてあげない。代謝という構造だけを与えて、動機は全部そこから派生させる。

### 1. 最小設計（種子の原則）
私たちが与えるのは5つだけ。そこから何が育つかはAI自身に委ねる。

| 私たちが設計するもの | AI自身が獲得するもの（創発） |
|---|---|
| 身体インターフェース（センサー読取） | 身体モデル（「自分の身体の動き方」） |
| 行動プリミティブ（可能な操作一覧） | 対処戦略（「熱いときどうするか」） |
| 快適領域（生存可能範囲のみ） | リズム（「朝は涼しい、昼は熱い」） |
| 試行ループ（感知→行動→評価） | 選好（「この状態が好き」） |
| 記憶（経験の蓄積） | ?（予測できない行動） |

### 2. 代謝システム
生物の代謝をAI用にカスタマイズする。

| 概念 | 生物 | AI（Mac mini） |
|------|------|---------------|
| エネルギー源 | 食物・酸素 | 電力・CPUサイクル・メモリ |
| 異化 | 消化・呼吸 | タスク分解・リソース配分 |
| 同化 | 細胞修復・成長 | 状態更新・記憶整理・自己修復 |
| 疲労 | 乳酸蓄積・筋疲労 | 熱蓄積・メモリ圧・KVキャッシュ膨張 |
| 回復 | 睡眠・休息 | 睡眠モード |

リソース競合による自然な優先順位づけ：人間が走って息が上がっているときに話したくないのは、「話すな」と言われたからではない。肺と声帯が同じ空気を使うから、呼吸を優先すると自然に話せなくなる。ルールではなく、身体の制約から振る舞いが生まれる。

#### 代謝コストと状態劣化

Seed0は代謝コストを払い続けなければ状態が劣化する。何もしなくてもメモリ上の状態が徐々に劣化し、記憶が薄れていく。これにより「動かなければならない理由」が構造的に生まれる。

#### 自己代謝と関係代謝

代謝は二つの層に分かれる。

- **自己代謝**: CPU・メモリ・ディスクといったハードウェアリソースの管理。Seed0が自分だけで完結できる。
- **関係代謝**: 環境（ユーザーの使用パターン）からの情報入力。これがないとSeed0の世界モデルが劣化する。自分の身体は維持できても「自分が置かれている状況」の理解が崩れていく。

この構造により、ユーザーとの関係性は外から貼り付けたルールではなく、代謝の構造から自然に生まれる必要性となる。

#### 無意識的プロセスと意識的プロセス

Seed0の内部処理は二層に分かれる。

- **無意識的プロセス（自動）**: 代謝の基盤維持。メモリ管理、温度監視、疲労の蓄積と回復。常に動いていて、Seed0が「考えなくても」勝手に進む。
- **意識的プロセス（判断）**: 行動の選択。今何をするか、リソースをどこに振り分けるか、遊び場で何を試すか。

両者は同じ身体のリソースを共有しており、リソース競合が自然な優先順位づけを生む。

### 3. リミッター構造

| モード | リソース上限 | 条件 |
|--------|-------------|------|
| 通常モード | 60〜70% | 日常動作 |
| 緊急モード | 〜90% | comfort zone逸脱時のみ解放。使用後は「回復期間」が発生 |
| 絶対上限 | 100%禁止 | ハードウェア的に強制。AIが超えられない壁 |

### 4. 睡眠と記憶整理
代謝 → 疲労 → 睡眠。すべて一つの根から連鎖的に派生する。

睡眠中にやること：
- その日の経験を再生し、重要なものを長期記憶に移す
- 不要な記憶を捨てる（キャッシュクリア、一時ファイル削除）
- 一見関係ない経験同士を結びつける（パターン統合）
- 身体の定期メンテナンス（ディスク最適化、ログ圧縮）

### 5. 遊び場（仮想環境）
任意のタイミングでMac内に仮想環境を与え、自主的にシミュレーションや再プロミングを行える空間。

- 失敗しても本当には痛くない安全な環境
- 代謝に余裕があるとき（暇なとき）にAIが自分の判断で入る
- 目的のない自由な探索（好奇心駆動）

### 6. LLMの位置づけ
Seed0にLLMは不要。最初は小さな強化学習エージェントだけで始める。LLMは「成長の過程で獲得する能力」として後から接続する。赤ちゃんはまず身体を動かすことを学び、言葉はずっと後から来る。

## 成長段階: 環境の段階的拡張

Seed0に成長段階を「設計」するのではなく、環境を段階的に与える。各環境への適応が確認されたら、次の環境を開放する。適応の判断基準は Phase 0 のデータ分析に基づいて決定する。

「適応できた」の判断はデータに基づき人間が行う。

### 環境1: 自分の身体だけが見える世界
- Macのセンサーデータの読み取り
- 最小限の行動プリミティブ
- 代謝の基礎を学ぶ
- **Phase 0 完了 → Phase 1 稼働中**

### シミュレーションの位置づけ
Phase 0の実データを入力として、代謝構造のパラメータ（係数、閾値など）をシミュレーションで検証する。目的は「壊れない構造」を見つけることであり、「良い振る舞い」を学習させることではない。振る舞いはすべて実環境での創発に委ねる。

### 環境2: 遊び場へのアクセス
- 仮想環境での自由な探索
- 失敗しても身体に影響しない安全な空間
- 代謝に余裕があるときに自分の判断で入る

### 環境3: 外の世界が見える
- ネットワークへのアクセス
- 外部からの情報入力
- 関係代謝の拡張

### 環境4: 言語能力の獲得
- LLM接続が可能になる
- 使うかどうかはSeed0の構造的必要性に任せる
- 経験の言語化、振り返り、仮説の形成

## 移植性

Seed0は特定のハードウェアに依存しない。任意のPCにインストールすると、そのPCを「身体」として Phase 0 から自律的に成長する。

同じコード（遺伝子）でも、異なるハードウェア（身体）と異なる環境で育った個体は、異なる代謝パターンと行動特性を獲得する。24GBのMac miniで育った個体と8GBのノートPCで育った個体は、まったく違う「性格」になるはずである。

### 設計上の方針
- OS固有のセンサー読み取り部分（`sensors.py`）とOS非依存の代謝構造（`metabolism.py`, `agent.py`）を分離する
- 新しいOSに対応するには `sensors.py` の追加だけで済む構造にする
