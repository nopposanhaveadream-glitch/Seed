# Seed0 — 代謝AI

> Mac miniを「身体」として生きる最小のAI

## これは何か

Seed0は、賢いAIでも強いAIでもない。**代謝AI**である。

Mac mini（M4 / 24GB）という「身体」を与えられ、その身体の有限性を感じ取り、自分自身で身体の使い方を学ぶAI。生物の代謝をそのままコピーするのではなく、代謝の本質的な構造だけを抽出し、AIの身体（Mac）に合った形にカスタマイズして組み込む。

## 核心仮説

**「有限性こそが知性の前提条件である」**

食べ物が無限にあれば狩りの戦略は生まれない。時間が無限にあれば優先順位という概念は意味をなくす。身体が壊れないなら慎重さも学ばない。AIに代謝という有限性を与えたとき、何が起きるか。これは実験である。

## 設計原則

### 1. 最小設計（種子の原則）
私たちが与えるのは5つだけ。そこから何が育つかはAI自身に委ねる。

| 私たちが設計するもの | AI自身が獲得するもの（創発） |
|---|---|
| 身体インターフェース（センサー読取） | 身体モデル（「自分の身体の動き方」） |
| 行動プリミティブ（可能な操作一覧） | 対処戦略（「熱いときどうするか」） |
| 快適領域（生存可能範囲のみ） | リズム（「朝は涼しい、昼は熱い」） |
| 試行ループ（感知→行動→評価） | 選好（「この状態が好き」） |
| 記憶（経験の蓄積） | ?（予測できない行動） |

### 2. 代謝システム
生物の代謝をAI用にカスタマイズする。

| 概念 | 生物 | AI（Mac mini） |
|------|------|---------------|
| エネルギー源 | 食物・酸素 | 電力・CPUサイクル・メモリ |
| 異化 | 消化・呼吸 | タスク分解・リソース配分 |
| 同化 | 細胞修復・成長 | 状態更新・記憶整理・自己修復 |
| 疲労 | 乳酸蓄積・筋疲労 | 熱蓄積・メモリ圧・KVキャッシュ膨張 |
| 回復 | 睡眠・休息 | 睡眠モード |

リソース競合による自然な優先順位づけ：人間が走って息が上がっているときに話したくないのは、「話すな」と言われたからではない。肺と声帯が同じ空気を使うから、呼吸を優先すると自然に話せなくなる。ルールではなく、身体の制約から振る舞いが生まれる。

### 3. リミッター構造

| モード | リソース上限 | 条件 |
|--------|-------------|------|
| 通常モード | 60〜70% | 日常動作 |
| 緊急モード | 〜90% | comfort zone逸脱時のみ解放。使用後は「回復期間」が発生 |
| 絶対上限 | 100%禁止 | ハードウェア的に強制。AIが超えられない壁 |

### 4. 睡眠と記憶整理
代謝 → 疲労 → 睡眠。すべて一つの根から連鎖的に派生する。

睡眠中にやること：
- その日の経験を再生し、重要なものを長期記憶に移す
- 不要な記憶を捨てる（キャッシュクリア、一時ファイル削除）
- 一見関係ない経験同士を結びつける（パターン統合）
- 身体の定期メンテナンス（ディスク最適化、ログ圧縮）

### 5. 遊び場（仮想環境）
任意のタイミングでMac内に仮想環境を与え、自主的にシミュレーションや再プロミングを行える空間。

- 失敗しても本当には痛くない安全な環境
- 代謝に余裕があるとき（暇なとき）にAIが自分の判断で入る
- 目的のない自由な探索（好奇心駆動）

### 6. LLMの位置づけ
Seed0にLLMは不要。最初は小さな強化学習エージェントだけで始める。LLMは「成長の過程で獲得する能力」として後から接続する。赤ちゃんはまず身体を動かすことを学び、言葉はずっと後から来る。

## 成長段階

### Phase 0 — 観察のみ
全センサーを読み取るが、行動は一切しない。数日かけて「身体が自然にどう動くか」のベースラインデータを蓄積する。

### Phase 1 — 砂場での学習
Phase 0のリアルデータから粗い身体モデルを構築し、シミュレーション内で「もしXをしたら？」を大量に試行する。

### Phase 2 — 見守りつきの身体化
実際のMacで行動を始めるが、安全な操作のみ。人間が拒否権を持つ。実結果と砂場の予測を比較し、身体モデルを修正する。

### Phase 3 — 自律的な身体化
安全範囲内で自己統治。継続的学習。砂場は「行動前に結果を想像する力」（想像力）として残り続ける。
