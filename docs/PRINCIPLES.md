# Seed0 設計原則

このプロジェクトのすべての設計判断は、以下の原則に従う。
迷ったときはここに立ち返る。

## 第一原則

**「Seed0に与えるのは構造だけ。動機はすべて構造から生まれなければならない。」**

## 派生原則

### 1. ルールではなく制約
「Xするな」「Yしろ」というルールは与えない。代わりに、構造的な制約を与える。制約の中でどう振る舞うかはSeed0自身が発見する。

### 2. 教えるのではなく環境を与える
「これが最適だ」と教えない。環境を与え、その環境に適応する過程で最適を自分で見つけさせる。

### 3. 段階的な環境拡張
適応が確認されてから次の環境を与える。未熟な段階で複雑な環境を与えない。

### 4. 有限性の保持
リソースの有限性はSeed0の知性の源泉である。有限性を取り除く最適化は行わない。

### 5. 代謝コストの存在
何もしなくても状態は劣化する。維持にはコストがかかる。これが自発的行動の動機になる。

### 6. 生物学的順序の尊重
身体（感覚器官）→ 神経系（反射と学習）→ 高次認知（言語と想像）。この順序を守る。

### 7. 関係性は設計しない
ユーザーとの関係性は、代謝構造から自然に生まれるのを待つ。外から定義しない。

### 8. 移植性
Seed0は特定のハードウェアに縛られない。任意の身体（PC）で育てることができる。
