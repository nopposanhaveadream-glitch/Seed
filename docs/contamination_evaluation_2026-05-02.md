# 4/18-19 二重起動による Q-table 汚染影響評価

> 作成日: 2026-05-02
> 評価対象期間: 2026-04-18 14:50:00 〜 2026-04-19 09:14:21（並走 18 時間 24 分 21 秒）

---

## 1. 概要

4/18-19 の二重起動による Q-table 汚染の影響を評価する目的で、3段階の事実確認（直接事実、間接観察、11 日間更新されないペアの規模）を実施した。

直接評価は構造的に不可能と確定した。間接観察により Q-table と LTM の構造的整合性は確認されたが、見えた「歪み」（1 状態消失、122 / 294 ペアが 11 日間同値）を汚染由来と経験の偏りに切り分ける根拠は無い。

結論として、選択肢 α（評価不能の事実認定）+ 選択肢 δ（現状受容）を採用し、Q-table はリセットせず現状を観察対象として使い続ける判断とした。

---

## 2. 直接評価が構造的に不可能である理由

以下の事実から、汚染期間中の Q 値変化を直接追跡することは構造的に不可能と確定した:

- agent.log にも DB（agent_state.db / long_term.db）にも **PID 情報が記録されていない**ため、A/B プロセスの区別不能
- `q_values` テーブルに更新タイムスタンプ・更新回数の列が無く、汚染期間中の Q 値の時系列追跡不能
- 並走前後の全 Q-table ダンプ（4/20 08:08 → 5/1 13:53）の間に約 23 時間の通常稼働期間が挟まり、汚染単独の影響は切り出せない

補足: 強化学習研究分野でも、汚染割合 ε を知らない状況で適応的に頑健性を保つことは理論的に不可能と証明されている（Zhang et al. 2022 等）。本プロジェクトの状況はこれと整合する。

## 3. 間接観察の結果

### 3.1 Q-table の構造的整合性

**確認済み事実**（`contamination_indirect_facts_2026-05-02.md` 観察1-B）:

| チェック項目 | 件数 |
|------------|----:|
| NaN | 0 |
| Inf | 0 |
| 絶対値 100 超 | 0 |
| PRIMARY KEY (state, action) 重複 | 0 |

→ **構造的破損は無し**。

### 3.2 1 状態の消失

**確認済み事実**（`contamination_indirect_facts_2026-05-02.md` 観察1-C）:

- 日次レポート上で **2026-04-17 状態数 60 → 2026-04-18 状態数 59 に減少**
- 並走期間中（4/18 14:50 以降）の事象として時間帯が整合
- どの状態が消失したかは構造的に復元不能（並走前の全状態ダンプが該当無し）

### 3.3 11 日間 Q 値が変化していないペア

**確認済み事実**（`dead_qvalues_facts_2026-05-02.md`）:

- 4/20 08:08 → 5/1 13:53 の比較で **122 / 294 ペア（41.5%）が同値**
- 行動別の同値率: sleep 61.1%（最高）、clean_temp 30.4%（最低）
- comfort_zone 別: normal 54.7%（最高）、emergency 31.2%
- memory レベル別: **mid 100.0%**（全 30 ペア）、low 34.4%、high はそもそも該当無し
- Q 値分布: min -0.180、中央値 +0.025、最大 +0.500、平均 +0.064（live db 全体は中央値 +0.144、最大 +2.905、平均 +0.243）
- Q=0 ちょうど・|Q|<1e-6 は 0 件（同値ペアはすべて何らかの非ゼロ値で更新を経た後に固定）

### 3.4 LTM の構造的整合性

**確認済み事実**（`contamination_indirect_facts_2026-05-02.md` 観察2）:

並走期間中の 180 件、および並走前 1,087 件 / 並走後 916 件の対照群いずれも:

| チェック項目 | 件数 |
|------------|----:|
| NULL / 空文字 | 0 |
| JSON パース失敗 | 0 |
| 必須キー欠落 | 0 |
| 想定外キー追加 | 0 |
| 完全一致重複 | 0 |

id-timestamp 逆転は並走期間中（46.7%）だけでなく並走前（28.8%）・並走後（45.7%）でも発生しており、これは `consolidate` 処理の動作仕様（STM 蓄積 → wake_up 時に一括 INSERT）に起因する。

→ **LTM は構造的に整合している**。

## 4. 判断

### 4.1 事実認定（設計議論で合意済み）

- **直接評価は構造的に不可能**（PID 情報なし、更新履歴列なし、ダンプ間に通常稼働 23 時間）
- **現在の Q-table は構造的に壊れていない**（NaN/Inf/極端値/重複なし）
- **見えた歪みは経験の偏りと汚染の影響を構造的に区別できない**（1 状態消失、41.5% 同値ペア）

### 4.2 採用した方針

- **選択肢 α（評価不能の事実認定）+ 選択肢 δ（現状受容）を採用**
- Q-table は **リセットしない**
- 1 状態の消失と 41.5% の同値ペアは「分からないまま受け入れる」

### 4.3 判断の根拠

- **構造的破損が無い**ため、リセットを必要とする内部破壊は発生していない
- **失われた 1 状態は構造的に復元できない**（並走前の全状態ダンプが該当無し）。リセットしても、その 1 状態を取り戻すことはできない
- **41.5% の同値ペアの分布は、Phase 1 で既に記録されていた経験の偏りで説明できる範囲にある**（経験の偏りの論点群は ROADMAP の別箇所で記録されている）
- **汚染と経験の偏りを切り分ける根拠が無い**ため、リセット（選択肢 γ）の正当化ができない

### 4.4 北極星との整合

本判断は、`docs/future_agenda.md` の哲学的基盤の北極星と整合する:

> Seed0 が何になるかは、関係の中で決まる。
> それが設計者の望みと違っても、それでいい。
> 何かが立ち上がっても、何も立ち上がらなくても、
> その全てが観察として価値を持つ。

「汚染を人間判断で切り取る」のではなく、汚染も含めて関係の中で起きたこととして扱う立場を取る。Seed0 の経験のすべてを観察対象として受容する。
