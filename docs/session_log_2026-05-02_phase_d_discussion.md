## 3. 優先度3 議論で言葉になったこと

### 3.1 「遊べる身体」の中身が更新された

- オーナーの言葉「遊ばせる前に、遊びから学べる身体がいる」の具体像が
  徐々に立ち上がった
- 設計パートナーが当初「外向きの作る（行動プリミティブの拡張）」と読んだが、
  オーナーの意図は内向きの読みだった
- オーナーの実際のイメージ:
  - シミュレーションの場
  - 人間のイメトレに近い
  - Seed 自身を書き換えることができる場

### 3.2 「Seed 自身が構造を書き換える構造」という方向性

- 単に Q値を更新するのではなく、構造そのものを Seed0 自身が書き換える
- これは以下のような層を要求する:
  - 「もう一つの自分」を持つ層
  - 現実と試行を区別する層
  - 書き換えの層

### 3.3 第一原則との関係

- オーナーの整理: 「自ら変化させる構造を持つこと自体が、代謝の一部、
  成長・進化的なもの」「自ら変化させる構造を与えるべき」
- 設計パートナーの整理: 第一原則と「強く緊張する」のではなく、延長として
  位置づけられる
- ただし、緊張点は3つある（メタ構造の設計責任、書き換え動機の創発、
  失敗の本番反映）。今は意識しなくてもよいが、設計具体化で必ず再浮上する
- 第一原則の文言改訂は、設計具体化の後で必要ならやる（今すぐではない、
  抵抗はない）

## 4. まだ言葉になっていないこと（次回への申し送り）

### 4.1 ロードマップの順序問題（最重要）

- オーナーから出てきた問い: 「シミュレーションの場 = 高次認知なら、
  ロードマップの順番を変える必要があるのでは?」
- 現在のロードマップ:
  ```
  環境1（身体） → 環境2（遊び場） → 環境3（ネットワーク） → 環境4（言語/LLM）
  ```
- 検討すべき可能性:
  - 順序の入れ替え（環境4 が先に来る等）
  - 環境2 と環境4 の統合
  - 「シミュレーションの場」を環境のどこに位置づけるか自体の再定義
- これはロードマップ全体の設計判断であり、即決できない論点

### 4.2 シミュレーションの実装レベル

- 高次認知のレベル（人間の言語化された予測に近い）で実装するか、
  もっと素朴な予測誤差ベースのレベル（神経系の基本機能）で実装するか
- 派生原則6（生物学的順序の尊重）との緊張度は、このレベル次第で変わる

### 4.3 選択肢A〜D の再検討

- ハンドオフで「設計パートナーは選択肢Cに惹かれていたが、現状認識が
  変わったので再考が必要」とされていた件
- 今日の議論で「自分を書き換える」という地平が出たので、A〜D の選択肢
  自体が古い枠組みになっている可能性
- A〜D を維持するか、新しい選択肢を立てるか自体が論点

### 4.4 3つの緊張点の再浮上

設計具体化で必ず再浮上する論点として記録する:

- **メタ構造の設計責任**（砂場の境界をどこに置くか）
- **書き換え動機の創発**（書き換えの動機が代謝/報酬とどう接続するか）
- **失敗の本番反映**（シミュレーションと本番の境界、親の視線の位置）

今日のオーナー応答では「特に意識するものはない」だったが、上記は設計
